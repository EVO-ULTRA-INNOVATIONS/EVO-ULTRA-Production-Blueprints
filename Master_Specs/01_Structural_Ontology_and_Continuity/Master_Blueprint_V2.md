## Executive Summary
Upgrading the EVO ULTRA AI Platform-as-a-Service (PaaS) into a living ecosystem of cognitive worker agents with autonomous self-modification capabilities.

## Pillar 1: Cognitive Microservices & Global Routing
-   **Global Delivery:** External App Load Balancer with Serverless NEGs to Cloud Run.
-   **Identity:** Google Cloud Identity Platform (CIAM) with `tenant_id` injection.
-   **Gateway:** `evo-gateway` as an Auth Interceptor.

## Pillar 2: The Tri-Memory Ontology
-   **Hot Memory (Firestore):** Real-time `agents`, `tasks`, and `sessions`.
-   **Deep Memory (BigQuery):** Immutable `deep_memory_ledger` for introspection.
-   **Cached Context (Vertex AI):** Persistent enterprise rules via Context Caching.

## Pillar 3: FinOps Strategy (Dual-Project)
-   **Project A (Cognitive Core):** Vertex AI Agent Builder & Search (Promotional Credits).
-   **Project B (Execution Engine):** Cloud Run, Firestore, BigQuery (Trial/Free Tiers).
-   **Circuit Breaker:** 95% budget alerts trigger global operation halt.

## Pillar 4: Autonomous Survival & Self-Tooling
-   **Quota-Aware Actuator:** Throttling concurrency to prevent 429s.
-   **Automated Rollbacks:** 5% error threshold triggers instant revision revert.
-   **Self-Tooling GitOps:** Architect Agent authorized to modify its own `Dockerfile`, commit, and redeploy.

## Engineering Notes
> **Engineer's Note (FinOps):** The "Dual-Project" split is essential for credit isolation. Mixing promotional GenAI credits with general compute usage leads to premature exhaustion of high-value AI tokens.
>
> **Engineer's Note (Self-Tooling):** The Architect Agent's ability to modify its own `package.json` requires strict validation. Implement a `pre-commit` hook that runs a security audit on any new dependencies.
